### Maven(实际企业开发)
传统的java项目或者web项目通常会使用很多的依赖jar包，通过新建libs目录，将jar包导入到libs中，但数量非常多，还要一个一个下载，如果某一个出现问题，整个项目可能就无法编译。   
所以通过Maven工具来实现jar包的管理，它会自动帮我们下载需要的jar包不再需要手动下载。
#### Maven功能概述
1) 清理：把之前项目编译的东西删除掉，为新的编译代码做准备。
2) 编译：把程序源代码编译为可执行文件，java-class文件
   批量的，maven可以同时把成百上千的文件编译为class
   javac不一样，javac一次编译一个文件
3) 测试：maven可以执行测试程序代码，验证你的功能是都正确。
   批量的，maven可以同时执行多个测试代码，同时测试很多功能。
4) 报告：生成测试报告。
5) 打包：把你的项目中所有的class文件，配置文件等所有资源放到一个压缩文件中。
   通常java程序压缩文件是jar扩展名的。对于web应用， 压缩文件扩展名是.war
6) 安装：将打包好的压缩文件安装到本地仓库。
7) 执行：安装好程序后就可以执行。
#### Maven自动下载jar包原理
本地仓库 私服仓库 中央仓库
1) 首先去本地仓库(本地计算机)查找该jar包，如果有则直接返回本地仓库的jar包
2) 如果本地仓库没有该jar包，则去中央仓库(国外网站)下载该jar包到本地仓库当中，后续再从本地仓库查找。
3) 为了解决中央仓库下载缓慢的问题，又搭建了一个私服仓库(企业内部)，本地仓库找不到该jar包后就去私服仓库查找，私服仓库找不到再去中央仓库下载到私服仓库。
   通过私服仓库便可实现不同开发者使用同一个仓库了。
#### Maven项目结构
java工程
1) src/main/java目录 存放java代码
2) src/main/resources目录 存放配置文件
3) src/test/java目录 存放测试代码
4) src/test/resources目录 存放测试配置文件
5) pom.xml 当前依赖的jar包
6) target目录 编译后的class文件

#### Maven常用命令
1) mvn clean 清除target目录下的编译文件
2) mvn compile 编译java代码
3) mvn test 执行测试
4) mvn package 打包文件并存放到项目的target目录下，打包好的文件通常都是编译后的class文件
5) mvn install 将打包好的文件安装在本地仓库中，可供其他项目使用。如果没有打包会先打包。
